Create `apps/web/src/components/demo/demo-player.tsx`:
- 'use client' component
- Reads `playerData`, `renderMode`, `theme`, `loop`, `charDelay`, `fps` from `useDemoStore`
- On mount (useEffect), dynamically import `@asciify/player` and call `registerAsciiPlayer()` to register the custom element. Guard with a ref to avoid double-registration. This is the SSR-safe pattern from Phase 5 (see ascii-canvas.tsx dynamic import pattern).
- Render an `<ascii-player>` custom element using a ref-based approach. Because React does not natively pass attributes to custom elements well, use a ref approach:
  1. Create a container div ref
  2. In useEffect, create the `<ascii-player>` element imperatively via `document.createElement('ascii-player')`
  3. Set attributes: `autoplay`, `controls`, `loop={loop}`, `fps={fps}`, `mode={renderMode}`, `theme={theme}`, `char-delay={charDelay}`, `width="800"`
  4. When `playerData` changes, set the element's `src` to a blob URL containing the JSON: `URL.createObjectURL(new Blob([JSON.stringify(playerData)], { type: 'application/json' }))`
  5. Append element to container div. On cleanup, remove element and revoke blob URL.
- When renderMode, theme, loop, charDelay, or fps change: update the element's attributes via `setAttribute()` in a useEffect that depends on these values.
- Style the container: rounded border, overflow hidden, bg-black, centered.

Create `apps/web/src/components/demo/demo-controls.tsx`:
- 'use client' component
- Reads/writes renderMode, theme, loop, charDelay from `useDemoStore`
- Render mode selector: three buttons (grid, proportional, typewriter) styled as a button group. Active mode gets `bg-primary text-primary-foreground`, others get `bg-muted`. Use the Button component from `@/components/ui/button`.
- Theme selector: buttons for each theme key from THEMES ('green-on-black', 'matrix', 'amber', 'white-on-black', 'blue'). Each button shows a small color swatch (8x8 rounded-full div with the theme's fgColor as bg) plus the theme name. Active theme highlighted.
- Loop mode: three buttons — 'forever', 'once', '3x'. Map 'forever' to 'forever', 'once' to 'once', '3x' to number 3.
- Char delay: Slider from `@/components/ui/slider` (min 10, max 200, step 5, default 30). Only visible when renderMode is 'typewriter'. Label: "Char delay: {charDelay}ms"
- Export test button: "Export HTML" button. On click:
  1. Read playerData from store
  2. Create a minimal self-contained HTML export:
     - Fetch `/player-bundle.js` (the IIFE bundle copied by the build script from Phase 5)
     - Build HTML: `<!DOCTYPE html><html><head><style>body{margin:0;background:#000}</style></head><body><script>${playerBundle}</script><ascii-player src="data:application/json,${encodeURIComponent(JSON.stringify(playerData))}" autoplay controls loop="${loop}" mode="${renderMode}" theme="${theme}" fps="${fps}" width="800"></ascii-player></body></html>`
     - Download via Blob + object URL, filename: `asciify-demo.html`
  3. Show Loader2 spinner while generating
- All controls in a vertical stack with section labels (h3 "Render Mode", h3 "Theme", etc.), Tailwind gap-4
