    ```typescript
    import { AsciiPlayer } from './player'
    import type { AsciiPlayerOptions, LoopMode, PlayerInputData } from './types'
    import { THEMES } from './types'
    ```

    **AsciiPlayerElement class** extends HTMLElement:

    ```typescript
    export class AsciiPlayerElement extends HTMLElement {
      static get observedAttributes() {
        return ['src', 'autoplay', 'loop', 'controls', 'fps', 'fg-color', 'bg-color', 'theme', 'font']
      }
    }
    ```

    **Constructor:**
    - `this._shadow = this.attachShadow({ mode: 'open' })` (open Shadow DOM per discretion)
    - Create `<canvas>` element, append to shadow root
    - Create a `<style>` element with `:host { display: block; width: 100%; }` and `canvas { display: block; width: 100%; }` plus `:host { --ascii-fg: #00ff00; --ascii-bg: #000000; }` defaults
    - Do NOT create player instance yet (wait for data)

    **connectedCallback():**
    - Read `src` attribute. If present, fetch JSON: `fetch(src).then(r => r.json()).then(data => this._initPlayer(data))`
    - If `this._pendingData` exists (set via `.data` property), call `this._initPlayer(this._pendingData)`

    **`.data` property setter** (for direct object assignment):
    ```typescript
    set data(value: PlayerInputData) {
      if (this.isConnected) {
        this._initPlayer(value)
      } else {
        this._pendingData = value
      }
    }
    ```

    **_initPlayer(data: PlayerInputData):**
    1. Destroy existing player if any: `this._player?.destroy()`
    2. Resolve colors with priority chain (per D-01):
       - Check `theme` attribute -> look up in `THEMES`
       - Check `fg-color` / `bg-color` attributes
       - Fall back to CSS custom properties: `getComputedStyle(this).getPropertyValue('--ascii-fg').trim()`
       - Final default: `'#00ff00'` / `'#000000'`
    3. Auto-size canvas (PLR-09):
       - Width: `this.clientWidth || 640`
       - Height: `Math.round(width * (data.metadata.rows / data.metadata.columns))`
       - Set `this._canvas.width = width; this._canvas.height = height` (buffer size, NOT CSS)
    4. Build options: `{ fps: Number(this.getAttribute('fps')) || data.metadata.fps, loop: parseLoop(this.getAttribute('loop')), autoplay: this.hasAttribute('autoplay'), font: this.getAttribute('font') || '14px monospace', fgColor, bgColor }`
    5. Create: `this._player = new AsciiPlayer(this._canvas, data, options)`
    6. Forward events: `this._player.addEventListener('timeupdate', e => this.dispatchEvent(new CustomEvent('timeupdate', { detail: (e as CustomEvent).detail, bubbles: true, composed: true })))`
    7. If `this.hasAttribute('controls')`, call `this._renderControls()`

    **_renderControls():**
    - Create a `<div>` with inline styles for a minimal controls bar (dark background, flex row)
    - Play/pause `<button>`: toggles `this._player.play()` / `this._player.pause()`. Text content toggles between 'Play' and 'Pause'.
    - Progress `<input type="range" min="0" max="100" value="0">`: on input, `this._player.seekTo(duration * value / 100)`
    - Listen to player `timeupdate` event to update range value: `range.value = String((currentTime / duration) * 100)`
    - Append controls div after canvas in shadow root

    **_resolveColor(attrName: string, cssVarName: string, defaultColor: string):**
    ```typescript
    private _resolveColor(attrName: string, cssVarName: string, defaultColor: string): string {
      const attr = this.getAttribute(attrName)
      if (attr) return attr
      const cssVar = getComputedStyle(this).getPropertyValue(cssVarName).trim()
      if (cssVar) return cssVar
      return defaultColor
    }
    ```

    **Helper parseLoop:**
    ```typescript
    function parseLoop(value: string | null): LoopMode {
      if (!value || value === 'once') return 'once'
      if (value === 'forever') return 'forever'
      const n = Number(value)
      return isNaN(n) ? 'once' : n
    }
    ```

    **attributeChangedCallback(name, oldVal, newVal):**
    - For 'src': if value changed, fetch new data and reinitialize
    - For 'controls': toggle controls visibility
    - For color/theme/font/fps/loop: if player exists, destroy and reinitialize with new options (simple approach — rebuild player on attribute change)

    **disconnectedCallback():**
    - `this._player?.destroy()`
    - `this._player = null`

    **Public API delegation** (PLR-04 on Web Component):
    - `play()`: `this._player?.play()`
    - `pause()`: `this._player?.pause()`
    - `seekTo(seconds)`: `this._player?.seekTo(seconds)`
    - `seekToFrame(index)`: `this._player?.seekToFrame(index)`
    - `get currentTime()`: `this._player?.currentTime ?? 0`
    - `get duration()`: `this._player?.duration ?? 0`

    **Registration** (per Pitfall 2 — guard against double registration):
    ```typescript
    export function registerAsciiPlayer(): void {
      if (typeof customElements !== 'undefined' && !customElements.get('ascii-player')) {
        customElements.define('ascii-player', AsciiPlayerElement)
      }
    }

    // Auto-register when module loads in browser
    registerAsciiPlayer()
    ```
